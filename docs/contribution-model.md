# Contribution model

New blueprints should follow the same information architecture:

1. Create the workload folder under the correct category.
2. Add Terraform and Bicep entry points that reference the shared AKS AVM baseline.
3. Add portal and `az` CLI guides.
4. Add workload-specific operational notes.
5. Add or extend a TechCommunity-ready blog package.

Use [`templates/workload-template`](../templates/workload-template) when onboarding a new project.
